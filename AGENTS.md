# AI 작업 지시

이 프로젝트에서 작업하는 AI는 `schoolp_ai_project_guide.md`를 반드시 먼저 읽고 따른다.

문서 내용이 충돌하면 더 제한적인 규칙을 따른다.

## 필수 규칙

- 로컬 실행 주소는 항상 `http://localhost:3000`이다. 3000번 포트가 사용 중이면 다른 포트로 우회 실행하지 말고 기존 dev server를 종료한 뒤 다시 실행한다.
- Windows에서 Node.js 또는 npm을 찾지 못하면 Node.js LTS 설치가 필요하다고 안내한다. Windows에서는 `start-windows.cmd`를 먼저 실행한다.
- 기존 프로젝트 전환 시 구조는 schoolp 기준으로 맞추되 기존 UI, 스타일, 레이아웃, 카피, 프론트 상호작용은 원본 수준으로 보존한다.
- 다운로드/안내 페이지는 프로젝트 구조와 개발 규칙 안내로만 사용한다.
- 기존 프로젝트 전환은 리디자인이 아니라 구조 이식 작업이다. 기본 화면은 원본 메인 화면과 동일해야 하며, 원본에 없던 UI는 실제 앱 화면에 추가하지 않는다.
- 사용자가 별도로 요청하기 전까지 포팅 과정에서 디자인을 재구성하지 않는다.
- 전환 결과물 안에 보존용 `legacy`, `backup`, `old` 폴더를 임의로 만들지 않는다.
- 기존 HTML/MD/static mockup은 최종 Next.js 화면과 문서에 필요한 내용만 이식하고, 중복 원본 파일은 결과물에 남기지 않는다.
- HTML/MD 정적 목업과 화면 중심 프로젝트도 이 Next.js 시작 파일을 사용한다.
- BFF와 로컬 DB는 필수가 아니다. 저장/조회/관리자 데이터가 필요한 경우에만 Route Handler, Prisma, SQLite를 사용한다.
- 기본 템플릿에는 활성 BFF/API/DB가 없다. 필요할 때만 `examples/optional-bff-db`를 참고해 실제 위치에 새로 만든다.
- 문서에 없는 API, 필드, 상태값, 인증 방식, 결제 방식, DB 구조를 임의로 만들지 않는다.
- 위험 주제를 다루는 mock UI 시안은 만들 수 있다.
- 실제 인증, 권한, 결제, 개인정보 판정, DB 저장, API 연동, 상태 변경, DB schema, 환경변수, 배포 설정은 임의로 수정하지 않는다.
- 불확실한 부분은 추측해서 구현하지 말고 `개발자 확인 필요`로 정리한다.
- 실제 API 연동은 `docs/API_CONTRACT.md`에 공식 API 문서가 있을 때만 진행한다.
- 기존 `.env.example` 복사와 기존 migration 실행은 가능하다. 환경변수 key 추가, DB schema 변경, migration 생성은 개발자 확인이 필요하다.
- Docker, CI/CD, 서버 배포 파일을 생성하거나 수정하지 않는다.
- Git, GitLab, branch, commit, push, remote 설정을 수행하지 않는다.
- FTP, SFTP, rsync, scp 등 전송 작업을 수행하지 않는다.
- package.json dependency를 임의로 추가하지 않는다.
- 작업 완료 시 수정한 파일, 실행한 명령, 확인한 화면/상태, 남은 질문을 요약한다.

## 작업 전 확인

작업을 시작하기 전에 아래를 판단한다.

```text
작업 유형:
위험 영역 포함 여부:
바로 진행 가능한 범위:
개발자 확인 필요 항목:
```

위험 영역이 포함되면 구현하지 말고 질문 목록으로 정리한다.
